# JWT Professional Notes

A comprehensive, production-oriented knowledge base for mastering JSON Web Tokens (JWTs), authentication architecture, security best practices, and modern identity patterns.

This repository is designed for developers, security engineers, architects, and technical leads who want to move beyond basic JWT usage and understand how to apply tokens effectively in real-world systems.

## Why this repository exists

JWTs are one of the most important building blocks in modern authentication systems, but they are also one of the most misunderstood. This project brings together:

- foundational concepts and terminology
- deep dives into JWT structure and claims
- signing and verification mechanics
- common attacks and defensive strategies
- enterprise-grade implementation patterns
- integration approaches for APIs, SPAs, mobile apps, GraphQL, WebSockets, and microservices

The goal is not only to explain JWTs, but to help you build secure, scalable, and maintainable authentication systems.

## What you will learn

This repository covers the full spectrum of JWT knowledge, including:

- the difference between authentication and authorization
- session-based vs token-based authentication
- JWT structure: header, payload, and signature
- registered claims and custom claims
- signing algorithms such as HMAC, RSA, and ECDSA
- JOSE and related standards
- validation and verification best practices
- algorithm confusion and other common security vulnerabilities
- token storage, XSS, CSRF, and browser security concerns
- expiration, revocation, refresh token rotation, and blacklisting
- key management and rotation strategies
- OAuth 2.0, OpenID Connect, and identity token concepts
- advanced topics such as mTLS, nested JWTs, JWE, JWKS, and edge validation

## Repository structure

The content is organized into a structured learning path:

- 00-prerequisites — core web and authentication foundations
- 01-fundamentals — what JWTs are and how they work
- 02-working-with-jwt — creation, verification, and debugging
- 03-advanced-claims-and-token-types — custom claims and token categories
- 04-security — attacks, vulnerabilities, and hardening
- 05-implementation-patterns — practical application patterns
- 06-sessions-and-state — hybrid and stateful authentication approaches
- 07-scaling-and-architecture — distributed and large-scale systems
- 08-integrations — framework and platform integrations
- 09-testing — validation and testing strategies
- 10-real-world-projects — applied examples and case studies
- 11-code-snippets — reusable implementation examples
- 12-resources-cheatsheets — quick references and study aids

## Learning path

For the best experience, follow the topics in order:

1. Start with the prerequisites and fundamentals.
2. Understand how JWTs are created and verified.
3. Learn about claims, tokens, and identity standards.
4. Study security risks and mitigation techniques.
5. Apply the concepts in real-world architectures and integrations.
